# Phase 1

## Goal

Establish a solid product foundation before investing in more advanced AI generation quality.

## Included in this phase

- Monorepo-style scaffold for frontend and backend
- SQLite schema and local seed data
- Dashboard focused on website and opportunity operations
- Websites CRUD
- Website Analysis page
- Opportunities CRUD
- Product and architecture documentation

## Explicitly not the focus yet

- High-quality AI article generation
- Polished automation logic
- CMS publishing integrations
- Billing
- Team management

## Delivery principle

Phase 1 is about building the operational shell of the product correctly:

1. Users can add and manage websites manually.
2. The system can store analysis-related website context.
3. Operators can review discovered opportunities and curate them directly.
4. The codebase is structured so later phases can add better AI and automation without a rewrite.
